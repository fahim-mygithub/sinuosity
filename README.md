# Sinuosity — WNY Scenic Ride Finder

Finds twisty, **scenic** backroads near Williamsville (36 Char Del Way), previews the
photo-worthy stops with Google imagery, and hands the route off to Google/Apple Maps for
navigation. Three modes:

- **Scenic** (default) — a slate of rides generated by an *agentic build-time pipeline*:
  candidate corridors are scored on a scenery rubric (curvature + viewpoints + greenery +
  water + notability) from real OpenStreetMap data, road-snapped with OSRM, given photo
  stop-points, and ranked by an automated judge panel. Each ride previews a satellite hero,
  a score breakdown, and a reviewable list of scenic stops with Street View thumbnails.
- **Curated** — hand-picked roads ranked by your twistiness/scenery/buzz weights.
- **Live scan** — real-time OSM curvature scan around home (hardened: multi-mirror retry,
  client timeout, distinct error states).

## Quick start

```bash
npm install
npm run dev      # http://localhost:5173
npm test         # unit tests (geometry, maps URLs, overpass, scoring)
npm run build    # production build into dist/  (base path /sinuosity/)
```

### Google Maps imagery (optional)

Street View + satellite thumbnails use a Google Maps Platform key. Copy `.env.example`
to `.env.local` and set `VITE_GOOGLE_MAPS_KEY` (enable *Maps Static API* + *Street View
Static API*; restrict the key to your Pages domain via HTTP referrers). **Without a key the
app still works** — scenic stops fall back to a placeholder plus keyless "Open in Street
View" deep-links.

## Regenerating the scenic dataset (the agentic pipeline)

The scenic routes in `src/data/scenicRoutes.ts` are generated, not hand-written:

```
scripts/gather-scenic.mjs     # 1. pull WNY scenic POIs from Overpass (+ OSRM check)
scripts/scenic-workflow.js    # 2. agentic compose (per corridor) -> judge panel -> rank
scripts/assemble-scenic.mjs   # 3. OSRM-snap geometry, measure curvature, emit the dataset
```

Step 2 runs via the multi-agent Workflow tool; steps 1 and 3 are plain Node. Re-run the
chain to refresh the slate. The pipeline grounds every route in real OSM features and
verifies stops with an adversarial judge panel before they ship.

## Architecture

```
src/
  lib/
    geometry.ts        haversine, cos-lat-corrected sinuosity, sharpest-turn (pure, tested)
    mapsUrl.ts         Google/Apple Maps URL builders, waypoint cap, NaN guards (tested)
    scoring.ts         curated composite scoring (tested)
    overpass.ts        hardened Overpass client: out-geom query, remark detection,
                       multi-mirror retry, abort support, name dedup (tested)
    scenicImagery.ts   Google Street View / satellite URL builders (runtime key) + deep-links
  data/
    types.ts           shared types (Route, ScenicRoute, ScenicStop, ScenicRubric)
    routes.ts          curated WNY route database
    scenicRoutes.ts    GENERATED scenic dataset (do not hand-edit)
  hooks/
    useLeafletMap.ts   map lifecycle, attribution, resize handling
    useBottomSheet.ts  pointer-driven sheet (mobile) / docked panel (desktop)
  lib/
    scenicMeta.ts      shared stop-kind icons + rubric labels
  components/
    RouteDetail.tsx        curated/scan detail + maps handoff
    ScenicRouteReview.tsx  full-page scenic review: satellite hero, rubric, big Street View stops
  App.tsx              wires it together (Scenic / Curated / Live scan tabs)
```

## Deploy (GitHub Pages)

`.github/workflows/deploy.yml` builds and deploys to Pages on push to `main`
(base path `/sinuosity/`). Set the optional `VITE_GOOGLE_MAPS_KEY` repo secret to enable
imagery on the live site. Enable Pages → Source: GitHub Actions in repo settings.

## What's real vs estimated

- **Curvature / sinuosity** — measured from real (OSRM-snapped or OSM-scanned) road geometry.
- **Scenery / greenery / water / notability** — derived from OSM POI density + Wikipedia/
  Wikidata tags near the route, then judge-reviewed. Higher fidelity than the old author
  estimates, but still a heuristic — verify pavement, season, and traffic before riding.
- **Stop imagery** — live Google Street View / satellite (with a key) at the stored coords.
